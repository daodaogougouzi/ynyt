# 易暖医途

## 项目简介

本项目是一个面向高校学生资助与心理关怀场景的微信小程序系统，覆盖学生端业务办理、AI 辅助咨询、心理关怀，以及老师端管理与审核等核心能力。

系统目标是将学生常用的困难认定、奖助申请、政策查询、心理测评、校园点滴、勤工助学等功能整合到一个微信小程序中，并通过本地 Mock API 与老师端页面形成可演示、可联调的完整闭环。

## 系统组成

项目当前主要由以下三部分组成：

### 1. 微信小程序学生端
位于 `miniprogram/`，是本项目的核心前端。

主要页面包括：
- 首页：公告、推荐项目、快捷入口、AI 使用向导
- AI 评估：困难认定预评估、认定申请、历史记录查看
- 学院奖助：奖助项目查询、详情查看、收藏、申请提交
- 心理关怀：SDS/SAS 自评、情绪记录、校园点滴、AI 树洞
- 我的：学生登录、学院绑定、成长树、历史记录、个人数据管理
- 扩展功能：临时补助、勤工助学、帮助中心、关于页、公告详情等

### 2. 本地 Mock API 服务
位于 `mock-api/`，用于提供小程序和老师端所需的后端接口。

主要职责包括：
- 提供首页、奖助项目、公告、学生、认定、申请等业务接口
- 保存认定申请、奖助申请、情绪事件、校园点滴等数据
- 提供 AI 对话接口与流式返回能力
- 托管老师端静态页面与上传资源

### 3. 老师端管理页面
位于 `teacher-web/`，由 Mock API 服务托管。

主要功能包括：
- 仪表盘与 AI 周期报告
- 学生管理与学生画像查看
- 困难认定审核
- 奖助项目维护与奖助申请审核
- 勤工岗位管理
- 高关怀提醒与关键词维护
- 截止提醒处理
- 校园点滴审核
- 首页公告维护

## 主要功能

### 学生端功能

#### 1. 首页门户与快捷分发
首页聚合公告、推荐项目、快捷功能入口和 AI 使用向导，承担整个系统的业务导航入口。

#### 2. 困难认定与 AI 评估
学生可以填写个人情况并进行困难认定预评估，随后提交正式认定申请，系统支持查看认定记录与审核状态。

#### 3. 学院奖助与申请管理
学生可以按类别浏览奖学金、助学金和资助项目，查看政策详情、资格匹配情况、截止时间，并提交申请材料。

#### 4. 收藏与历史记录
系统支持收藏奖助项目、查看评估历史、查看申请记录，方便学生追踪已办理事项。

#### 5. 心理关怀中心
包含每日暖心语、减压指引、SDS/SAS 量表测评、校园点滴投稿、情绪事件记录，以及 AI 树洞聊天等功能。

#### 6. AI 助手
系统提供材料草稿生成与通用 AI 对话能力，支持流式回复展示，并在高风险情绪场景下提供求助提示。

#### 7. 勤工助学与临时补助
学生可以浏览勤工岗位、查看岗位要求并提交申请，也可以查看临时补助相关说明与办理指引。

### 老师端功能

#### 1. 学生资助审核与管理
老师可以查看学生认定申请、奖助申请、困难等级与资助记录，并进行审核处理。

#### 2. 公告与奖助项目维护
支持维护首页公告、封面与附件，也支持调整奖助项目开放状态、金额展示、资格要求等信息。

#### 3. 心理关怀运营
老师端聚合高关怀提醒、情绪关键词、学生心理趋势和校园点滴审核等能力，用于开展心理支持与干预。

#### 4. AI 辅助运营
老师端支持 AI 周期报告和 AI 助手能力，用于总结阶段数据、生成建议和辅助工作决策。

## 实现原理

## 1. 技术架构

本项目采用“微信小程序前端 + 本地 Node.js Mock API + 老师端静态页面”的结构：

- 小程序前端：原生微信小程序 + TypeScript + Sass
- 数据接口：通过 `wx.request` 访问 Mock API 或远程 API
- 本地存储：使用 `wx.setStorageSync` 保存登录态、收藏、历史记录等数据
- 老师端：基于原生 HTML/CSS/JavaScript 构建，由 Mock API 直接托管

项目未使用 Taro、uni-app 或 React/Vue 类前端框架，而是采用原生微信小程序页面与组件开发模式。

## 2. 登录与身份传递

学生登录成功后，前端会将当前学生会话信息缓存到本地存储。后续请求会自动从本地读取当前学生 ID，并通过请求头 `X-Student-Id` 传递给服务端，从而实现按学生维度的数据隔离和业务处理。

## 3. 数据访问机制

项目在 `miniprogram/utils/api.ts` 中统一封装了请求方法：
- 普通接口使用标准 JSON 请求
- AI 对话场景支持流式返回
- 当后端返回 401 时会自动引导用户重新登录

此外，前端允许通过本地存储中的 `xjt_api_base_url` 动态覆盖默认接口地址，以支持联调不同环境。

## 4. 困难认定与推荐逻辑

困难认定相关功能采用“两层逻辑”：

- 一层是前端本地规则，用于进行快速预估和演示
- 一层是后端接口驱动的正式认定申请与审核流程

学生填写家庭、成绩、获奖等信息后，系统会根据规则给出预估结果，并可继续上传附件、提交正式申请。

## 5. 奖助申请流程

奖助学金模块会先拉取学生信息、奖助列表、资格条件和截止提醒，再在详情页根据项目规则判断学生是否具备申请资格。学生提交申请后，老师端可继续审核。

## 6. AI 对话与流式响应

AI 聊天相关能力通过后端接口提供，前端支持按 SSE 风格分块接收内容并实时渲染回复。这样可以提升交互体验，避免长文本一次性返回造成等待时间过长。

服务端同时预留了 Claude 相关环境变量，支持接入 Anthropic 模型进行 AI 对话与老师端 AI 报告能力。

## 7. 心理关怀机制

心理关怀模块通过量表评分、负面情绪记录、关键词预警和 AI 聊天共同构成一个多维关怀体系：
- 量表用于标准化测评
- 情绪事件用于连续记录
- 关键词用于风险分级提醒
- AI 树洞用于即时陪伴式交互

## 开发环境要求

### 基础环境

建议准备以下环境：

- Node.js（建议使用较新的 LTS 版本）
- npm
- 微信开发者工具
- 可用的小程序开发者账号 / AppID（项目配置中已存在一个 AppID）

### 关键说明

1. 项目运行核心依赖微信开发者工具，不能像普通 Web 项目一样直接在浏览器中运行学生端。
2. 项目默认 API 地址为：
   - `https://ynyt.nat100.top`
3. 如果需要本地联调，可改为本地 Mock API 地址，例如：
   - `http://127.0.0.1:3100`
4. 老师端页面由 Mock API 服务托管，启动服务后即可在浏览器访问。

## 安装与启动

### 1. 安装依赖

在项目根目录执行：

```bash
npm install
```

### 2. 启动本地 Mock API

#### 方式一：直接使用批处理脚本（Windows）

```bash
./start-mock-api.bat
```

该脚本会：
- 将服务端口设置为 `3100`
- 将公开访问地址设置为 `http://127.0.0.1:3100`
- 启动 `mock-api/server.js`

#### 方式二：手动启动

在 Bash 中可使用：

```bash
XJT_MOCK_API_PORT=3100 XJT_PUBLIC_BASE_URL=http://127.0.0.1:3100 node mock-api/server.js
```

如果是在 Windows PowerShell 中，可使用：

```powershell
$env:XJT_MOCK_API_PORT="3100"
$env:XJT_PUBLIC_BASE_URL="http://127.0.0.1:3100"
node .\mock-api\server.js
```

启动成功后，可访问：

- Mock API：`http://127.0.0.1:3100`
- 老师端：`http://127.0.0.1:3100/teacher`

老师端演示账号为：

- 账号：`李老师`
- 密码：`123456`

### 3. 启动微信小程序

1. 打开微信开发者工具
2. 选择“导入项目”
3. 项目目录选择当前仓库根目录
4. 小程序根目录会自动识别为 `miniprogram/`
5. 安装依赖后编译运行

如果要让小程序改用本地 Mock API，请确保前端请求基地址指向：

```text
http://127.0.0.1:3100
```

项目代码中默认支持通过本地存储覆盖 API 基地址，因此联调时可以按实际开发需要进行配置。

## 部署说明

## 1. 学生端部署

学生端本质上是微信小程序，部署方式不是传统 Web 服务器发布，而是通过微信开发者工具上传代码并发布。

### 发布流程

1. 在微信开发者工具中完成代码检查与预览
2. 确认小程序业务配置、服务器域名、上传文件无误
3. 点击“上传”生成体验版
4. 在微信公众平台进行版本提交与审核
5. 审核通过后发布线上版本

### 部署前检查项

- 确认 `project.config.json` 中的 AppID 是否为目标小程序
- 确认服务器域名已在微信公众平台配置为合法域名
- 确认小程序请求地址、上传地址、下载地址均可访问
- 确认基础库版本满足项目需求（项目配置为 `2.32.3`）
- 确认 `skyline`、TypeScript、Sass 编译能力在开发工具中可用

## 2. Mock API / 演示服务部署

如果要部署演示环境或本地联调环境，可将 `mock-api/server.js` 作为 Node.js 服务运行。

### 本地部署命令

```bash
npm install
XJT_MOCK_API_PORT=3100 XJT_PUBLIC_BASE_URL=http://127.0.0.1:3100 node mock-api/server.js
```

### 生产 / 服务器部署示例

```bash
npm install --production
XJT_MOCK_API_PORT=3100 \
XJT_PUBLIC_BASE_URL=https://your-domain.example.com \
node mock-api/server.js
```

部署后：
- API 接口路径位于 `/api/*`
- 老师端页面路径位于 `/teacher`
- 上传文件路径位于 `/uploads/*`

## 3. AI 能力部署

如果需要启用真实 Claude AI 能力，需要在启动服务前配置以下环境变量：

```bash
ANTHROPIC_API_KEY=your_api_key
ANTHROPIC_BASE_URL=https://api.anthropic.com
ANTHROPIC_MODEL=claude-sonnet-4-6
CLAUDE_MAX_OUTPUT_TOKENS=4096
CLAUDE_TEMPERATURE=0.3
CLAUDE_API_TIMEOUT_MS=30000
CLAUDE_API_RETRY_COUNT=1
```

完整启动示例：

```bash
ANTHROPIC_API_KEY=your_api_key \
ANTHROPIC_BASE_URL=https://api.anthropic.com \
ANTHROPIC_MODEL=claude-sonnet-4-6 \
XJT_MOCK_API_PORT=3100 \
XJT_PUBLIC_BASE_URL=https://your-domain.example.com \
node mock-api/server.js
```

- 校园点滴数据

## 注意事项

1. 仓库当前没有在 `package.json` 中定义 `dev`、`build`、`start` 等 npm scripts，因此运行方式以手动命令和微信开发者工具为主。
2. 学生端依赖微信开发者工具与微信运行时能力，如文件选择、图片选择、文档打开等，无法在普通浏览器中直接运行。
3. 老师端依赖 Mock API 提供静态资源与接口，因此不能单独双击 HTML 文件运行完整功能。
4. 如需接入真实后端或正式环境，建议统一替换 API 基地址，并按实际业务改造 Mock 数据与接口逻辑。
